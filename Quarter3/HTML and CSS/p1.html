HTML Basics

  1. HTML stands for hypertext markup language. Hypertext is text that is not constrained to be linear. Markup just indicates that it is for annotations and not for the primary content of a page.
  2. It is not a programming language, it is only a markup language, as it adds context and structure to text.
  3. One can use an online IDE like Cloud9 to work on the file and then preview the file live.
  4. In an HTML page:
    • The title is displayed in the tab part of the tab of the browser that the page is open in.
    • The body is displayed below that, in the main part of the tab.
  5. The following tags, in actuality, have no spaces between the angle brackets and other characters.
    • < h1 > is a tag that indicates heading 1. Other headings are possible, and by default, heading 1 is the largest.
    • < p > indicates a paragraph. Nearby text is seperated from it by a n empty line above and below whatever is enclosed by the tags.
    • < br > does not need a closing tag. It makes a line break.
  6. Image elements:
    • src refers to URL of the image being displayed. The characters come from the word source.
    • alt is text that is displayed if the image cannot be, for some reason.
    • width is the width of the image.
    • height is, as expected, the height of the image.
  7. The < a > element defines a hyperlink. The href attribute specifies the URL that the hyperlink links to.
  8. A local link links to a page on the same website. A remote link does not. Local links can be specified by the text that occurs after the base URL. If the link if found on https://www.example.com/intro.html, it can reference https://www.example.com/works.html simply by "works.html" for the href attribute. Remote links, on the other hand, must include the entire URL, including https://.
  9. Example is at the top of this page.

HTML Page Structure

  1. Nested HTML elements are HTML elements inside others, for instance, an image inside of a list element, as seen above by the winged heart that is in the sixth element of the list above. Another example are the nested lists --a unordered list within an ordered list element.
  2. There is a table above with the title, "More Flash Comics".
    • < tr > defines a row of a table.
    • < td > defines "table data", with the default being left aligned text with no special formatting.
    • < th > defines a table heading, with the default being centered, bolded text.
  3. This can be seen throughout the page.
  4. Block-level elements take up the width of the page and the height of the content, while inline content only takes up as much width as the content.
  5. < div > and < span > are for organizing page and grouping different areas together, for instance, for aesthetics and formatting. < div > is a block-level element and < span > is inline.
  6. For any formatting or aesthetics, < div > and < span > should be used, as a table shouls be used only when making a data table and not for formatting as I have used it for above.
