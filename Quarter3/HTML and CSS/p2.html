Styles
  1. CSS stands for cascading style sheets. It is used to style a webpage and add functionality.
    • external CSS is in a separate file
    • internal CSS is in < style > tags in the < head >. It overrides external CSS.
    • inline CSS is used by using the < style > attribute and it will override both other forms of CSS.
  3. Class is used to style a group of elements all in the same way, for instance, while id is used for one particular element.
  4. You will find examples of these in the code.
  5. Likewise with this question about color.
  6. The background of the page has plenty examples of different properties.
  7. The header utilizes several property/value pairs.
  8. The em unit is relative to the current font size, while px refers to pixels which is an absolute length. Pixels do, however, very from divice to device in regards to how many pixels are actually used.
  9. Four hyperlink states: link (unvisited link), visited, hover, active (the moment a link is clicked) These are customized by:
    a:[name of state] {
    /*tab*/[customization]
    }
Page Structure


  1. The box model describes how CSS elements and content are formatted. Text and images are the content. Then there is a clear area around content that is the padding. A border goes around the padding and then there is a clear margin that is outside of the border.
  2. There are borders and outlines on the top fixed heading bar and both text boxes here in the body.
  3. There are also margins specified using both shorthand and long form.
  4. Pixels are absolute forms of measurement, meaning they are displayed at the same size across all devices. Relative dimensions on the other hand change in relation to the viewing window.
  5. By default, a fixed size div does not have its overflow clipped. It is visible outside of the element's box. The overflow property allows one to change it so that the content is clipped or that it is clipped with the addition of a scroll bar.
  6. I actually see no reason why one should ever use absolute dimensions if they are aiming to have a nice looking webpage. Aesthetics and general visibility of content is improved by relative dimensions.
  7. Block and inline formatting can be specified under the 'display' attribute. Block elements start on a neine and take up the full width available, while inline elements do not start on a new line and only take up the amount of space required to display the content.
  8. Float allows text to wrap around images, and clear specifies which sides of an element are not allowed to float.
  9. Floated elements that exceed the width of their container are not visible.
  10. The width of the element is specified, and then putting 'auto' as the value for the property 'margin' evenly splits the remaining space between the element and the visible space.
